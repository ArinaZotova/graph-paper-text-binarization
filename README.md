# Graph Paper Text Binarization

Курсовая работа по дисциплине **«Методы и средства обработки изображений»**.

Проект реализует алгоритм бинаризации рукописного текста на изображениях графической бумаги. Алгоритм подавляет клетчатую структуру листа и формирует бинарную маску текста.

## Описание задачи

На вход подается изображение рукописного текста на листе графической бумаги. Необходимо получить бинарную маску, в которой белым цветом выделен только рукописный текст, а фон и линии клетки остаются черными.

Для оценки качества может использоваться эталонная маска, размеченная вручную.

Основная сложность задачи связана с тем, что линии графической бумаги являются регулярной клетчатой структурой и могут ошибочно выделяться как часть текста. Кроме того, реальные фотографии могут содержать наклон листа, перспективные искажения и неравномерное освещение.

## Используемые технологии

- C++17;
- OpenCV;
- CMake.

## Данные для тестирования

Для проверки работы программы используется папка `data`.

В ней находятся:

- `FIRST.png` — исходное изображение рукописного текста на графической бумаге;
- `FIRST_TARGET.png` — эталонная бинарная маска, размеченная вручную.

После запуска программы результаты сохраняются в папку `results`.

## Основные этапы алгоритма

1. Загрузка исходного изображения.
2. Предварительная обработка: выделение рабочей области, обрезка и приведение к единому размеру.
3. Перевод изображения в градации серого.
4. При необходимости нормализация освещения.
5. Применение быстрого преобразования Фурье.
6. Подавление частот, соответствующих клетчатой структуре.
7. Обратное преобразование Фурье.
8. Бинаризация методом Оцу.
9. Дилатация и очистка связных компонент.
10. Расчет метрик качества при наличии эталонной маски.

## Частотная фильтрация

Алгоритм использует быстрое преобразование Фурье, так как клетчатая структура графической бумаги является регулярным повторяющимся паттерном.

В частотной области линии клетки проявляются в виде выраженных компонент, связанных с горизонтальными и вертикальными направлениями. Для их подавления используется частотная маска в форме креста. При этом центральная область низких частот сохраняется, чтобы не удалить основную информацию об изображении.

Такой подход наиболее корректно работает, когда линии графической бумаги расположены близко к горизонтальному и вертикальному направлениям. Поэтому для реальных изображений с заметным наклоном может потребоваться предварительное перспективное выравнивание рабочей области.

## Метрики качества

В проекте рассчитываются следующие метрики:

- IoU;
- Precision;
- Recall;
- F1-score.

Для итогового эксперимента на основном реальном изображении были получены следующие значения:

| IoU | Precision | Recall | F1-score |
|---:|---:|---:|---:|
| 0.6532 | 0.9012 | 0.7037 | 0.7903 |

## Дополнительный эксперимент с наклоном

Для анализа влияния наклона линий графической бумаги был проведен дополнительный эксперимент на реальном изображении, где лист расположен под углом к границам кадра.

Алгоритм запускался в двух вариантах:

1. без перспективного выравнивания рабочей области;
2. после перспективного выравнивания рабочей области.

Полученные метрики:

| Вариант | IoU | Precision | Recall | F1-score |
|---|---:|---:|---:|---:|
| Без перспективного выравнивания | 0.0740 | 0.0748 | 0.8637 | 0.1377 |
| После перспективного выравнивания | 0.5710 | 0.6671 | 0.7986 | 0.7269 |

Эксперимент показывает, что перспективное выравнивание существенно уменьшает количество ложноположительных выделений и повышает устойчивость частотной фильтрации на реальных изображениях.

## Сборка проекта

Для сборки проекта необходимо выполнить следующие команды:

```bash
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

## Запуск программы

Общий формат запуска программы:

```bash
./demo <input_image> <output_dir> <gt_mask> --visualize
```

В данном эксперименте использовались следующие пути:

- `input_image`: `../data/FIRST.png`;
- `output_dir`: `../results`;
- `gt_mask`: `../data/FIRST_TARGET.png`.

Для Windows аналогично запускается файл `demo.exe` из папки `Release`.

## Результаты работы

После запуска программы результаты обработки сохраняются в папку `results`.

При использовании флага `--visualize` сохраняются промежуточные и итоговые изображения, по которым можно проверить ход обработки:

- изображение после предварительной обработки;
- результат обратного преобразования Фурье;
- предсказанная бинарная маска текста;
- карта ошибок при наличии эталонной маски.

Если эталонная маска передана при запуске, программа также выводит в консоль значения метрик качества: IoU, Precision, Recall и F1-score.

## Примечание

Основная программная реализация выполняет бинаризацию изображения и расчет метрик качества. Дополнительный эксперимент с перспективным выравниванием использовался для анализа влияния наклона линий графической бумаги на качество частотной фильтрации.
